# AGENTS.md

## 项目定位
- 本目录是“基于 Vue3 + SpringBoot 的博物馆文物综合管理平台”前端子项目。
- 这是毕业设计项目，优先保证页面可运行、演示稳定、交互直观、易于答辩。
- 不要过度设计，优先选择简单、直接、容易维护的实现方案。

## 前端技术栈
- Vue 3
- Vue CLI 5
- JavaScript
- ESLint

## 作用范围
- 只允许读取、创建、修改、删除 `mhmp` 目录内的文件。
- 不要修改后端目录 `mhmp_back` 中的任何文件。
- 除非用户明确说明，否则不要跨目录修改文件。
- 如果需求涉及后端联调，只能处理前端页面、接口对接、请求参数和返回结构适配，不要直接修改后端代码。

## 前端规则
- 按当前项目现有结构开发，优先沿用 `src/main.js`、`src/App.vue`、`src/components`、`src/assets` 的组织方式。
- 不要随意引入 `Vue Router`、状态管理库、UI 组件库或当前项目中不存在的新依赖。
- 不要随意修改 `package.json`、`vue.config.js`、脚手架配置和构建方式。
- 页面实现优先简单、直观、好演示，不追求复杂动效和重型封装。
- 组件拆分适度即可，不要为小页面过度抽象。
- 保持命名风格、目录风格、代码风格与现有前端项目一致。
- 代码注释适量，重点说明业务规则和不直观的交互逻辑。

## 实现优先级
- 优先复用现有代码和组件结构，不顺手重构整个前端。
- 修改范围尽量小，不要影响无关页面和功能。
- 优先保证页面可访问、功能可演示、交互可讲解。
- 除非用户明确要求，否则不要引入复杂状态流、权限前端框架或大型样式体系。

## 工作方式
- 修改前先分析相关页面、组件和资源引用关系。
- 需求不明确时，优先选择与当前项目风格一致、最简单可运行的前端实现。
- 如果发现已有改动与当前任务冲突，不要擅自覆盖或回退，先判断影响范围。
- 接口联调时，优先兼容现有后端返回结构和字段命名。

## 输出要求
- 完成任务后请明确说明：
- 新增或修改了哪些文件
- 前端启动步骤
- 已完成功能
- 下一步建议
