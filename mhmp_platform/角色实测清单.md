## 3. 按设计的角色边界

| 角色 | 设计定位 |
| --- | --- |
| `admin` | 全量系统管理 + 全量业务办理 + 全量审批 |
| `senior_researcher` | 研究员全部业务 + 入库审批 + 出库审批/驳回 + 修复审批 + 修复验收 + 转存派发 + 可给研究员派发盘点 |
| `researcher` | 文物建档、入库登记、出库申请、库存查询、发起/处理本人盘点、处理本人转存、修复申请与过程记录；不应拥有审批类入口与按钮 |
| `docent` | 保持讲解/只读定位，不应拥有系统管理权限，也不应拥有审批、派发、建档删除等高风险按钮 |

## 6. 列表检索专项清单

### 6.1 文物列表

- [ ] 关键字支持按文物编号、名称检索。
- [ ] 类别、材质、保存状态、当前状态、库位筛选可正常联动。
- [ ] 列表中应能看清编号、名称、类别、材质、保存状态、当前状态、库位等核心信息。

### 6.2 库存查询

- [ ] 关键字支持按 `文物编号 / 名称 / 年代 / 来源` 检索。
- [ ] 类别、材质、保存状态、当前状态、库位筛选可正常联动。
- [ ] 列表必须展示“保存状态”列。
- [ ] 检索后分页、重置、页码切换正常。

### 6.3 发起转存

- [ ] 关键字支持按 `文物编号 / 名称 / 年代 / 来源` 检索。
- [ ] 类别、材质、保存状态、库位筛选可正常联动。
- [ ] 已存在未完成转存任务的文物不应出现在候选列表里。
- [ ] 列表中应能看清类别、材质、保存状态、当前库位。

### 6.4 出库申请 / 出库审批 / 出库归还

- [ ] 关键字支持按 `单号 / 用途 / 去向 / 经手人` 检索。
- [ ] 状态筛选切换正常。
- [ ] `出库归还` 页默认只显示当前经手人本人负责的出库单。
- [ ] 不同经手人登录后，出库归还列表应随账号切换而变化。

### 6.5 修复相关列表

- [ ] 修复过程上传的图片名称过长时应省略显示，不应挤破卡片布局。

## 7. 状态联动专项清单

### 7.1 入库

- [ ] 发起入库后，相关文物状态应进入“入库待审批”。
- [ ] 入库审批通过后，文物状态应变为“在库”，并写入入库库位。

### 7.2 出库

- [ ] 发起出库后，相关文物状态应变为“出库待审批”。
- [ ] 出库审批通过后，文物状态应变为“已出库”。
- [ ] 出库审批驳回后，文物状态应恢复为“在库”。
- [ ] 由当前经手人在“出库归还”登记后，文物状态应恢复为“在库”。

### 7.3 转存

- [ ] 发起转存任务时，文物当前库位不应立即改变。
- [ ] 研究员在“我的转存”确认后，文物库位才应切换为目标库位。
- [ ] 确认后，该转存任务应变为 `COMPLETED`。

### 7.4 盘点

- [ ] 新建盘点任务后，初始状态为 `CREATED / 未开始`。
- [ ] 首次保存盘点明细后，任务状态应自动变为 `IN_PROGRESS / 进行中`。
- [ ] 提交盘点任务后，任务状态应变为 `COMPLETED / 已完成`。

### 7.5 修复

- [ ] 发起修复申请后，应进入修复申请/审批链路。
- [ ] 修复审批通过后，文物状态应进入“修复中”。
- [ ] 修复过程更新后，任务状态和日志记录应实时刷新。
- [ ] 申请验收后，任务进入待验收状态。
- [ ] 验收通过后，文物状态应回到“在库”，并写回修复后的保存状态与返库位置。
- [ ] 验收确认弹窗内不应再出现“驳回”按钮，驳回应只保留在列表操作中。

## 11. 本轮清单的判断标准

- 菜单不对：先看数据库角色菜单绑定是否正确。
- 按钮不对：再看按钮权限码是否正确下发。
- 列表查不到：先看筛选条件是否符合最新逻辑，再看当前账号是否有权限。
- 状态不联动：优先判定为业务规则缺陷，需结合文物详情页时间线和数据库状态一起核对。
